### 本程序用于模拟发送锐捷心跳包。

> 可以用来间接处理某些学校mentohust验证阶段无法通过的用户。
>
> 使用虚拟机或者其他电脑，连接完成之后利用一些连接信息，即可模拟锐捷在线。
>
> 锐捷是通过mac判断（其中之一）是否要将你的包交付出去的。

```
beat
本程序要锐捷拨号前运行
然后将自动捕获锐捷认证成功信息作为心跳包依据
锐捷拨号成功后，直接**杀死**锐捷进程
然后按下Ctrl+C通知本程序开始模拟发送心跳包
***如果程序已在后台运行，会直接结束后台程序，然后退出程序***

----------------------------------Usage---------------------------------------

-w	--windows		表明是在windows环境下，默认在非windows环境下
				该参数已废弃
-i	--interface		指定上网的网卡名称
-e	--echointerval		心跳包间隔(单位:秒)(默认30)
-b	--background		成功运行后后台运行程序
-u	--updatemac		更新指定网卡的mac(会重启网卡)
-d	--debug			调试模式
-h	--help			显示该信息


手动指定必要信息，来使得程序不等待抓取信息，立即开始模拟发送心跳包
***需要同时指定下面四个参数***
-k	--echokey
-n	--echono
-r	--remotemac
-s	--sourcemac

--------------------------------注意事项----------------------------------------

该程序自行计算提供的echokey似乎有一两位有问题，所以当发送256个包之后会断开连接
所以这里建议把心跳包间隔调大一点，广金的大概是6分钟检测一次把目前

    echointerval(s)    activetime
    30                 128m
    60                 256m
    113                8h
    256                18.2h
    300                21.3h


```
